# Відстань Хемінга і сума відстаней

## Що порівнюється

Кожен експерт задає власну трійку пріоритетів:

1. перше місце
2. друге місце
3. третє місце

Генетичний алгоритм будує повне ранжування всіх об'єктів. Для порівняння з конкретним експертом із цього повного ранжування беруться тільки ті об'єкти, які входять до його трійки, зі збереженням їхнього порядку.

## Відстань Хемінга

Після цього маємо два списки однакової довжини:

- порядок експерта
- порядок, отриманий з хромосоми

Відстань Хемінга дорівнює кількості позицій, у яких ці списки відрізняються.

Приклад:

- експерт: `A > B > C`
- хромосома після проєкції: `A > C > B`

Порівняння:

- позиція 1: `A = A` -> 0
- позиція 2: `B != C` -> 1
- позиція 3: `C != B` -> 1

Отже, відстань Хемінга дорівнює `2`.

## Сума відстаней Хемінга

Нехай для однієї хромосоми відстані до експертів дорівнюють:

- `d1`
- `d2`
- `d3`
- ...

Тоді сума відстаней обчислюється так:

`sumDistance = d1 + d2 + d3 + ...`

У поточній реалізації саме ця сума є основною цільовою величиною.

## Fitness-функція

У проєкті використано обернену формулу:

`fitness = 1 / (1 + sumDistance)`

Тобто:

- чим менша сума відстаней Хемінга, тим більше значення `fitness`
- якщо `sumDistance = 0`, то `fitness = 1`
- генетичний алгоритм відбирає хромосоми з найбільшим `fitness`, що відповідає мінімальній сумі відстаней

## Як це працює в коді

1. Хромосома задає повний порядок об'єктів.
2. Для кожного експерта з цього порядку витягується його трійка.
3. Обчислюється відстань Хемінга між експертною трійкою та трійкою з хромосоми.
4. Усі індивідуальні відстані додаються.
5. Отримана сума записується як `sumDistance`.
6. Значення `fitness` обчислюється як `1 / (1 + sumDistance)`.

## Приклад

Нехай хромосома дає порядок:

`A > C > B > D > E`

Експерти мають такі трійки:

- експерт 1: `A > B > C`
- експерт 2: `C > A > D`
- експерт 3: `B > D > A`

### Експерт 1

Проєкція хромосоми:

`A > C > B`

Порівняння з `A > B > C`:

- `A = A` -> 0
- `C != B` -> 1
- `B != C` -> 1

Отже:

`d1 = 2`

### Експерт 2

Проєкція хромосоми:

`A > C > D`

Порівняння з `C > A > D`:

- `A != C` -> 1
- `C != A` -> 1
- `D = D` -> 0

Отже:

`d2 = 2`

### Експерт 3

Проєкція хромосоми:

`A > B > D`

Порівняння з `B > D > A`:

- `A != B` -> 1
- `B != D` -> 1
- `D != A` -> 1

Отже:

`d3 = 3`

### Загальний результат

`sumDistance = 2 + 2 + 3 = 7`

`fitness = 1 / (1 + 7) = 0.125`

## Короткий висновок

У цій версії алгоритму використовується тільки сума відстаней Хемінга. Чим менша сумарна відмінність хромосоми від експертних трійок, тим кращою вважається хромосома і тим більшим є значення `fitness`.
