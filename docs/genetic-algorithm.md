# Як виконується генетичний алгоритм у проєкті

## Призначення алгоритму

Генетичний алгоритм використовується для пошуку найкращого порядку об'єктів після того, як на етапі ЛР2 уже було сформовано фінальну підмножину кандидатів.

Ідея така:

- маємо набір об'єктів після фільтрації
- маємо експертні трійки з ЛР1
- потрібно знайти таке колективне ранжування, яке найкраще узгоджується з експертами

У поточній реалізації узгодженість оцінюється через суму відстаней Хеммінга.

## Де це реалізовано

Основна логіка знаходиться в `app/admin/page.tsx` у блоці `lab2GeneticAnalysis`.

Алгоритм спирається на такі допоміжні функції:

- `projectRankingToExpertChoices`
- `calculateHammingDistance`
- `calculateSumHammingDistance`
- `calculateGeneticFitness`
- `rotateChromosome`
- `crossoverChromosomes`
- `mutateChromosome`
- `deduplicatePopulation`

## Основні сутності

### Хромосома

Хромосома — це один варіант упорядкування об'єктів.

Наприклад:

`A > C > B > D`

У поточній версії хромосома подається просто як масив назв об'єктів типу `string[]`.

### Популяція

Популяція — це набір хромосом, які розглядаються одночасно в одному поколінні.

### Покоління

Покоління — це один цикл роботи генетичного алгоритму:

1. оцінювання всіх особин
2. відбір кращих
3. створення нових особин
4. формування нової популяції

У проєкті зараз використовується `4` покоління.

## Крок 1. Формування кандидатів

На початку алгоритм бере фінальну підмножину після ЛР2:

- з `lab2Analysis.finalSubset`

Із неї формується список кандидатів.

## Крок 2. Формування експертних трійок

Далі з голосів експертів будуються `expertRows`.

Для кожного експерта беруться:

- `first_place`
- `second_place`
- `third_place`

Але з них залишаються тільки ті об'єкти, які входять до фінальної підмножини кандидатів.

Тобто якщо частина об'єктів відсіялась на ЛР2, вони вже не беруть участі в генетичному алгоритмі.

## Крок 3. Побудова початкової популяції

Алгоритм не стартує з однієї хромосоми. Він відразу створює початкову популяцію з кількох різних порядків.

У початковий пул входять:

- початковий порядок кандидатів
- зворотний порядок кандидатів

Після цього додаються ще варіанти через циклічний зсув:

- `rotateChromosome(defaultOrder, shift)`
- `rotateChromosome(reverseOrder, shift)`

Також додаються кілька мутованих стартових рішень:

- `mutateChromosome(defaultOrder, 1)`
- `mutateChromosome(reverseOrder, 2)`

Після створення всіх цих варіантів однакові хромосоми прибираються через `deduplicatePopulation`.

## Крок 4. Як рахується відстань Хеммінга

### Проєкція глобального ранжування на експертну трійку

Хромосома задає повний порядок усіх кандидатів.

Щоб порівняти її з конкретним експертом, потрібно взяти з хромосоми тільки ті об'єкти, які входять до трійки цього експерта.

Це робить функція:

`projectRankingToExpertChoices`

Наприклад:

- хромосома: `A > C > B > D > E`
- експертна трійка: `A > B > C`

Після проєкції з хромосоми беремо тільки `A`, `B`, `C`, але в тому порядку, як вони стоять у хромосомі:

`A > C > B`

### Обчислення відстані Хеммінга

Далі функція `calculateHammingDistance` порівнює:

- експертний порядок
- порядок після проєкції

по позиціях.

Кожен незбіг додає `1`.

У прикладі:

- експерт: `A > B > C`
- проєкція: `A > C > B`

Маємо:

- позиція 1: `A = A` -> 0
- позиція 2: `B != C` -> 1
- позиція 3: `C != B` -> 1

Отже:

`distance = 2`

## Крок 5. Як рахується сума відстаней Хеммінга

Функція `calculateSumHammingDistance` проходить по всіх експертах і додає їхні індивідуальні відстані:

`sumDistance = d1 + d2 + d3 + ... + dn`

де:

- `d1` — відстань до першого експерта
- `d2` — відстань до другого експерта
- ...

Чим менше значення `sumDistance`, тим краще хромосома узгоджується із сукупною думкою експертів.

## Крок 6. Fitness-функція

У функції `calculateGeneticFitness` використовується:

`fitness = 1 / (1 + sumDistance)`

Це зроблено для того, щоб:

- уникнути ділення на нуль
- зробити більший `fitness` ознакою кращої особини
- при меншій сумі відстаней отримувати більше значення fitness

Приклад:

- якщо `sumDistance = 0`, то `fitness = 1`
- якщо `sumDistance = 3`, то `fitness = 0.25`
- якщо `sumDistance = 7`, то `fitness = 0.125`

Тобто при зростанні помилки fitness зменшується.

## Крок 7. Оцінювання популяції

У кожному поколінні кожна хромосома:

1. перетворюється на звичайний список об'єктів
2. отримує `sumDistance`
3. отримує `fitness`

Після цього вся популяція сортується.

Поточне правило сортування:

- спочатку за найменшим `sumDistance`
- якщо сума однакова, тоді за більшим `fitness`

Оскільки `fitness` вже є функцією від `sumDistance`, друге правило фактично слугує технічним тайбрейкером.

## Крок 8. Відбір кращих особин

Після сортування беруться `elites` — найкращі особини покоління.

Їхня кількість визначається як:

- або не менше 2
- або приблизно третина популяції

Саме вони стають основою для побудови наступного покоління.

## Крок 9. Схрещування

Функція `crossoverChromosomes` бере двох батьків:

- `firstParent`
- `secondParent`

Механіка така:

1. Обирається точка розрізу `pivot`.
2. Початок першого батька копіюється в дитину.
3. Із другого батька додаються всі елементи, яких ще немає в дитині.

Приклад:

- батько 1: `A > B > C > D`
- батько 2: `C > A > D > B`
- `pivot = 2`

Початок від першого батька:

`A > B`

Із другого батька додаються відсутні елементи в його порядку:

- `C`
- `D`

Результат:

`A > B > C > D`

Такий підхід зберігає порядок без дублювання об'єктів.

## Крок 10. Мутація

Функція `mutateChromosome` змінює хромосому локально:

1. обираються дві позиції
2. елементи на цих позиціях міняються місцями

Це потрібно, щоб алгоритм не застряг у вузькій області пошуку і міг досліджувати нові варіанти.

## Крок 11. Ротація

Функція `rotateChromosome` циклічно зсуває послідовність.

Приклад:

- було: `A > B > C > D`
- зсув на 1: `B > C > D > A`

Це ще один спосіб створювати нові особини, не руйнуючи повністю структуру хромосоми.

## Крок 12. Прибирання дублікатів

Після генерації нових хромосом можуть з'явитися однакові варіанти.

Функція `deduplicatePopulation`:

- переводить кожну хромосому у строку вигляду `A|B|C|D`
- залишає тільки перше входження кожного унікального порядку

Це підтримує різноманітність популяції.

## Крок 13. Побудова нового покоління

Нова популяція формується так:

1. у неї переходять `elites`
2. далі додаються діти після схрещування
3. додаються мутовані варіанти
4. додаються ротації

Якщо після видалення дублікатів популяція вийшла замалою, алгоритм добирає додаткові варіанти через ротації та мутації стартових порядків.

## Крок 14. Повторення циклу

Цей процес повторюється для кожного покоління.

У вашому коді зараз:

- кількість поколінь: `4`
- цільовий розмір популяції: `40`

На кожному кроці зберігаються:

- усі особини покоління
- найкраща особина покоління

Це потім відображається в адмін-панелі.

## Що вважається найкращою хромосомою

Найкраща хромосома — це та, яка має:

- найменшу суму відстаней Хеммінга до експертів

А отже:

- найбільше значення `fitness`

## Короткий приклад проходу

Нехай є 4 об'єкти:

`A, B, C, D`

І 2 експерти:

- експерт 1: `A > B > C`
- експерт 2: `B > C > D`

Одна з хромосом:

`A > C > B > D`

Для експерта 1:

- проєкція: `A > C > B`
- експерт: `A > B > C`
- відстань: `2`

Для експерта 2:

- проєкція: `C > B > D`
- експерт: `B > C > D`
- відстань: `2`

Отже:

`sumDistance = 2 + 2 = 4`

`fitness = 1 / (1 + 4) = 0.2`

Інша хромосома може дати, наприклад:

`sumDistance = 2`

Тоді:

`fitness = 1 / (1 + 2) = 0.3333`

Отже друга хромосома буде кращою.

## Висновок

Генетичний алгоритм у цьому проєкті не перебирає всі перестановки напряму, а покроково покращує популяцію варіантів.

Його логіка така:

1. створити набір стартових хромосом
2. оцінити їх за сумою відстаней Хеммінга
3. вибрати кращі
4. породити нові через схрещування, мутацію і зсув
5. повторити кілька поколінь

Мета алгоритму — знайти такий порядок об'єктів, який найкраще узгоджується з експертними трійками.
